# データ分析の流れ

このドキュメントでは、データ分析の **「型」** を、本プロジェクトのファイルがどの段階に対応しているかと併せて整理します。

---

## 全体像

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ ①データ取得  │ → │ ②EDA         │ → │ ③前処理      │ → │ ④モデル学習  │ → │ ⑤評価        │
│              │    │ (探索)       │    │ + 特徴量生成 │    │              │    │              │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                                                          ↓
                                                                                  ┌──────────────┐
                                                                                  │ ⑥チューニング│
                                                                                  └──────────────┘
```

**ポイント**: 矢印は実は一方通行ではなく、評価結果を見て前処理を見直したり、新しい特徴量を作ったりと **何度も往復します**。

---

## ①データ取得

> 「**まずデータを手元に揃える**」フェーズ。

| やること | 本プロジェクトのファイル |
|-|-|
| 元データのダウンロード | `src/data_loader.py` の `load_raw()` |
| `data/raw/` に保存 | `data/raw/california_housing.csv` |

### コツ

- 元データは **一切加工せずに** `data/raw/` に置く
- API やデータベースからの取得スクリプトもここに書く

---

## ②EDA (探索的データ解析)

> 「**そのデータがどんな性質か**」を理解するフェーズ。  
> ここを丁寧にやるとモデル精度が大きく変わります。

| やること | 本プロジェクトのファイル |
|-|-|
| 形状, 型, 欠損の確認 | `notebooks/01_EDA.ipynb` |
| 基本統計量 | 同上 (`df.describe()`) |
| 相関係数とヒートマップ | 同上 (`df.corr()`, `sns.heatmap`) |
| 各変数の分布 | 同上 (`df.hist()`) |
| 目的変数の特徴 | 同上 (5.00001 で上限処理されている) |

### コツ

- 「気づき」をマークダウンセルに書き残す
- 後の前処理 / 後処理のアイデアになる

> 例: 「Price が 5.00001 で頭打ち」と気づけば、予測値もクリッピングするアイデアが出る。

---

## ③前処理 + 特徴量生成

> 「**モデルが食べやすい形に整える**」フェーズ。
> 業界では「**データ分析は前処理が 8 割**」と言われます。

### 代表的な処理

| 処理 | 本プロジェクトの実装 |
|-|-|
| 欠損値処理 (削除・補完) | `src/preprocessing.fill_missing` |
| カテゴリ変数のエンコーディング | (今回は不要だが one-hot / Label encoding) |
| スケーリング (標準化/正規化) | `src/preprocessing.scale_features` |
| 特徴量生成 | `src/features.add_household_features` |

### 特徴量生成の例

世帯数 = `Population / AveOccup` という ドメイン知識を使って新しい列を作る:

```python
df['Household'] = df['Population'] / df['AveOccup']
df['AllRooms'] = df['Household'] * df['AveRooms']
df['AllBedrms'] = df['Household'] * df['AveBedrms']
```

### コツ

- 学習用と検証用で **同じ前処理** を適用する
- 検証用に学習用の `Scaler` を使い回す (新しく fit してはいけない)

---

## ④モデル学習

> 「**データからパターンを学ばせる**」フェーズ。

### サブステップ

1. **データ分割**: 学習用 (80%) / 検証用 (20%) — `train_test_split`
2. **モデル選択**: 線形回帰? 木系? ニューラルネットワーク?
3. **学習実行**: scikit-learn は `model.fit(X, y)` だけ

### 本プロジェクトでの対応

| やること | ファイル |
|-|-|
| 学習用 / 検証用に分割 | `src/modeling.split_train_test` |
| 線形回帰モデル | `src/modeling.train_linear_regression` |
| XGBoost モデル | `src/modeling.train_xgboost` |
| モデルの保存 | `src/modeling.save_model` → `models/*.pkl` |

---

## ⑤評価

> 「**学習したモデルがどのくらい予測できるか**」を測るフェーズ。

### 代表的な指標 (回帰)

| 指標 | 意味 |
|-|-|
| MAE  | 平均絶対誤差 |
| MSE  | 平均二乗誤差 |
| RMSE | MSE の平方根 (単位が元データと揃う) |
| R²   | 決定係数 (1 に近いほど良い) |

### 本プロジェクトでの対応

| やること | ファイル |
|-|-|
| 評価指標の計算 | `src/evaluation.regression_metrics` |
| まとめて表示 | `src/evaluation.print_metrics` |
| ノートブックでの可視化 | `notebooks/04_evaluation.ipynb` |

---

## ⑥チューニング

> 「**指標を見て改善する**」フェーズ。

### 代表的な手法

| 手法 | やること |
|-|-|
| **後処理** | 予測値を実際の範囲にクリップ etc |
| **ハイパーパラメータ探索** | グリッドサーチ, ベイズ最適化 |
| **特徴量選択 / 追加** | 不要な列を消す, 新しい列を増やす |
| **アンサンブル学習** | バギング / ブースティング / スタッキング |

### 本プロジェクトでの対応

| やること | ファイル |
|-|-|
| 予測値のクリッピング | `src/modeling.predict_with_clipping` |
| XGBoost (ブースティング) との比較 | `notebooks/04_evaluation.ipynb` |

## まとめ — 「型」を覚えると速い

データ分析は華やかなテクニックよりも、

1. **データの素性を把握する**
2. **モデルが学べる形に整える**
3. **学習・評価・改善のループを回す**

という地味な作業の積み重ねです。
この **型** に沿って動けると、初めて触るデータでも迷わず進められます。

このプロジェクトは、その型を **ディレクトリ構造として体に染み込ませる** ことを狙ったテンプレートです。
